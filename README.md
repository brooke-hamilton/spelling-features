# spelling-features

A collection of [Dev Container Features](https://containers.dev/implementors/features/) for spell checking tools.

## Features

### `pyspelling`

Installs [pyspelling](https://facelessuser.github.io/pyspelling/), a spell checking tool powered by [Aspell](http://aspell.net/).

The following packages are installed:

- `aspell` and `aspell-en` (system packages via apt)
- `pyspelling` (installed via [pipx](https://pypa.github.io/pipx/))

#### Usage

Add the feature to your `devcontainer.json`:

```json
{
    "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
    "features": {
        "ghcr.io/brooke-hamilton/spelling-features/pyspelling:1": {}
    }
}
```

### `cspell`

Installs [cspell](https://cspell.org/), a spell checker for code.

The following packages are installed:

- `nodejs` (installed via NodeSource if not already present)
- `cspell` (installed globally via npm)

#### Usage

Add the feature to your `devcontainer.json`:

```json
{
    "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
    "features": {
        "ghcr.io/brooke-hamilton/spelling-features/cspell:1": {}
    }
}
```

## Prerequisites

- [Docker](https://www.docker.com/)
- [Node.js](https://nodejs.org/)
- [Dev Container CLI](https://github.com/devcontainers/cli)

Install the Dev Container CLI:

```sh
make install-devcontainer-cli
```

## Testing

Run the full test suite:

```sh
make test
```

Run only autogenerated tests (validates installation across multiple base images):

```sh
make test-autogenerated
```

Run only scenario tests:

```sh
make test-scenarios
```

Run tests for a specific feature:

```sh
make test-pyspelling
make test-cspell
```

Quick single-image test for faster iteration:

```sh
make test-pyspelling-quick
make test-cspell-quick
```

### Test types

- **Autogenerated tests** (`test.sh`) run against auto-generated `devcontainer.json` files using the feature with default options. These are executed against `debian:latest`, `ubuntu:latest`, and `mcr.microsoft.com/devcontainers/base:ubuntu`.
- **Scenario tests** (`scenarios.json` + matching `.sh` files) test specific option combinations or multi-feature setups.

## Linting

Lint all shell scripts with [ShellCheck](https://www.shellcheck.net/):

```sh
make lint
```

## Publishing

Features are published to GitHub Container Registry (GHCR) using the [Dev Container Publish Action](https://github.com/devcontainers/action).

To publish:

1. Go to **Actions** > **Release dev container features & Generate Documentation**.
2. Click **Run workflow** on the `main` branch.

The workflow will:

- Package and publish each feature to `ghcr.io/brooke-hamilton/spelling-features/<feature>`.
- Generate documentation and open a PR with any updates.

> **Note:** GHCR packages are private by default. To allow public access, go to the package settings and set visibility to **public**.
